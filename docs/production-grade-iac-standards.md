# 生产级 IaC 项目标准体系

## Context

ClawForce 项目的 `infra/` 目录已从 PoC 阶段逐步成长——具备了良好的 Construct 分层设计、93 项测试、CDK Nag 合规检查和完善的规范文档体系。但距离"生产级"仍有关键缺口：**无 CI/CD 流水线、密钥硬编码、无多环境支持、告警无通知、无漂移检测**。

本文档定义一套完整的生产级 IaC 标准体系，覆盖从 0 到 1 的全生命周期，并基于 AWS Well-Architected Framework、DORA 指标、CIS 基准等权威框架。每个标准项附带 **成熟度等级（L1-L5）**，团队可根据业务阶段渐进式达标。

---

## 成熟度等级定义

| 等级 | 名称 | 含义 | 适用阶段 |
|------|------|------|---------|
| **L1** | 手动 | 有 IaC 代码但手动操作为主 | PoC / 原型验证 |
| **L2** | 可重复 | 自动化测试 + 基础安全 + 手动部署 | 内部工具 / 早期产品 |
| **L3** | 自动化 | CI/CD 流水线 + 多环境 + 密钥管理 | **生产最低门槛** |
| **L4** | 可治理 | Policy-as-Code + 漂移检测 + 合规审计 | 企业级 / 合规要求 |
| **L5** | 持续优化 | FinOps + DORA 度量 + 自愈 + 混沌工程 | 大规模 / 高可用 |

> **生产级最低要求 = L3**。L1-L2 仅适合开发/原型阶段。

### IaC 测试金字塔

```
          /\
         /  \        E2E 测试（部署完整环境 + 冒烟测试）
        /E2E \       10-60 min, 仅生产前
       /------\
      /集成测试 \     部署真实资源，验证交互
     /----------\    5-30 min, 合并到 main 时
    /  合约测试   \   模块接口兼容性
   /--------------\  1-5 min, 每个 PR
  / 策略测试       \  CDK Nag / OPA / Checkov
 /------------------\ 1-5 min, 每个 PR
/ 单元测试           \ CDK Assertions / 快照测试
/--------------------\ 秒级, 每次提交
/  静态分析 / Lint     \ ESLint + Prettier + cdk synth
/------------------------\ 秒级, 每次提交
```

## ClawForce 项目现状评估

基于以上标准，当前 `infra/` 项目的成熟度评估：

| 领域 | 当前等级 | 关键依据 | 达到 L3 需要 |
|------|---------|---------|-------------|
| **Construct 设计** | **L2+** | 良好分层、readonly Props、合理默认值 | 已基本达标 |
| **测试** | **L2+** | 93 项测试、CDK Nag、Fine-grained Assertions | 补充快照测试、覆盖率门槛强制 |
| **安全** | **L2** | CDK Nag、IMDSv2、EBS 加密、WAF | **密钥迁移到 Secrets Manager**、启用 CloudTrail/VPC Flow Logs |
| **代码质量** | **L2+** | TS strict、ESLint+Prettier、现代工具链 | 补充 pre-commit hooks |
| **监控** | **L1+** | Log Groups + CPU/状态告警 + CW Agent | **SNS 告警通知**、Dashboard |
| **CI/CD** | **L1** | 纯手动部署 | **实现 GitHub Actions 流水线** |
| **多环境** | **L1** | 单环境、无 dev/staging/prod 区分 | **实现环境参数化** |
| **成本** | **L1** | GP3、t3.medium，无标签/预算 | **实现标签策略 + 预算告警** |
| **密钥管理** | **L1** | Gateway Token 硬编码在 constants.ts | **迁移到 SSM/Secrets Manager** |
| **文档** | **L2+** | 9 份详尽的 .claude/rules/ 文档 | 自定义 README.md |
| **漂移检测** | **L1** | 无 | L4 要求 |
| **DORA 度量** | **L1** | DevPace 有基础度量 | 需要与 CI/CD 集成 |

**综合评估：L2（可重复）**——已有良好的代码基础和测试覆盖，但缺乏自动化部署和生产级安全措施。

### 优先级路线图（L2 → L3）

1. **P0 — 密钥管理**：Gateway Token 迁移到 SSM Parameter Store（安全风险）
2. **P0 — CI/CD 流水线**：实现 GitHub Actions（lint → test → synth → diff → deploy）
3. **P1 — 告警通知**：SNS Topic + 订阅（告警有人收到）
4. **P1 — 多环境支持**：环境参数化（dev/prod 从同一代码部署）
5. **P1 — 资源标签**：全局标签策略实施
6. **P2 — 快照测试 + 覆盖率门槛**：补全测试金字塔
7. **P2 — Pre-commit hooks**：格式 + 密钥扫描 + lint
8. **P2 — CloudWatch Dashboard**：可视化运维
