## 意图

- **用户原话**：将 PoC 手动部署转为 CDK TypeScript 自动化，涵盖 EC2+SG+IAM+EBS 全栈，内嵌所有 PoC 修复项 <!-- source: user -->
- **范围**：<!-- source: claude, 从 PoC 报告和迭代规划推断 -->
  - 做什么：创建 AWS CDK TypeScript 项目，定义 EC2、IAM Role/Instance Profile、Security Group、EBS 构建体；User Data 脚本嵌入所有 PoC 修复项（IMDSv2 hop limit=2、Ubuntu 24.04 ssh.service、ASCII-only descriptions）；Docker Compose override 配置（AWS_REGION 注入）；CfnOutput 连接信息
  - 不做什么：不包含 ALB/HTTPS/WAF（属于 PF-003）、不包含 CloudWatch 监控（属于 PF-004）、不修改 OpenClaw 源码
- **验收条件**：<!-- source: claude -->
  1. Given CDK 项目结构已创建, When 运行 `npx cdk synth`, Then 生成有效 CloudFormation 模板无报错
  2. Given 所有构建体已定义, When 检查合成的模板, Then 包含 EC2 Instance、IAM Role、Instance Profile、SecurityGroup、EBS Volume 资源
  3. Given PoC 修复项已嵌入, When 检查 User Data 脚本, Then 包含 IMDSv2 hop limit=2、ssh.service（非 sshd）、ASCII-only 描述
  4. Given Docker Compose override 已配置, When 检查 User Data, Then 包含 AWS_REGION 注入和 Bedrock Inference Profile 格式模型 ID
  5. Given 参数化配置已定义, When 检查 Stack props, Then 所有可配置项（instanceType、region、allowedIp 等）通过 props 参数化
- **方案**：<!-- source: user+claude, 用户选择 CDK TypeScript -->
  - 使用 AWS CDK TypeScript — 与 OpenClaw Node.js 生态统一，L2 构建体自动处理最佳实践
  - 单 Stack 结构：ClawForceStack 包含所有资源（首迭代不拆分多 Stack）
  - User Data 使用 ec2.UserData 类和 Asset 管理，PoC 修复项作为构建体参数
- **约束**：
  - AWS CDK >= 2.x，Node.js >= 18
  - 区域默认 us-east-1（与 PoC 一致）
  - 暂不考虑多环境（首迭代单 Stack，不引入 Stage/Pipeline）

### 执行计划

1. **`infra/`**：CDK TypeScript 项目初始化
   - 预期：`cdk init app --language typescript` 生成项目骨架，含 bin/、lib/、package.json、tsconfig.json、cdk.json
   - 命令：`ls infra/bin/ infra/lib/`

2. **`infra/lib/constructs/networking.ts`**：Security Group 构建体
   - 预期：定义 SSH(22)、Gateway(18789)、Control UI(18790)、Browser(18791) 端口规则，ingress 限制为参数化 IP
   - 命令：`cd infra && npx tsc --noEmit`

3. **`infra/lib/constructs/iam.ts`**：IAM Role + Instance Profile + Bedrock 策略构建体
   - 预期：EC2 assume role、Bedrock InvokeModel 权限（含 inference-profile/* ARN）、自动绑定 Instance Profile
   - 命令：`cd infra && npx tsc --noEmit`

4. **`infra/lib/constructs/compute.ts`**：EC2 实例 + EBS + IMDSv2 + User Data 构建体
   - 预期：t3.medium 默认实例、30GB gp3 加密 EBS、IMDSv2 强制（HttpTokens.REQUIRED, hopLimit=2）；User Data 嵌入 Docker 安装 + OpenClaw 部署 + ssh.service 加固 + 全部 PoC 修复项
   - 命令：`cd infra && npx tsc --noEmit`

5. **`infra/lib/clawforce-stack.ts`**：主 Stack 组装 + CfnOutput
   - 预期：Stack 集成所有构建体；CfnOutput 输出 instanceId、publicIp、sshCommand、gatewayUrl
   - 命令：`cd infra && npx cdk synth`

6. **验证**：cdk synth 端到端 + PoC 经验覆盖检查
   - 预期：CloudFormation 模板生成成功；grep 验证 6 条 PoC 经验修复项全部嵌入
   - 命令：`cd infra && npx cdk synth | grep -c 'HttpTokens\|ssh.service\|inference-profile'`
