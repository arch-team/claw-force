## 意图

- **用户原话**：推进 PF-002 Bedrock 适配，CR-001 已覆盖 Inference Profile 格式和 IAM 策略，剩余模型发现能力 <!-- source: user -->
- **范围**：<!-- source: claude, 从 PF-002 定义和 CR-001 溢出分析推断 -->
  - 做什么：预部署 Bedrock 模型验证脚本（检查目标区域模型可用性）；cdk.json 模型预设配置（常用 Bedrock 模型 Inference Profile 格式）；NPM validate 脚本集成
  - 不做什么：不修改 OpenClaw 源码；不创建 CDK Custom Resource（避免过度设计）；不实现运行时模型切换（属于 PF-002 后续 CR）
- **验收条件**：<!-- source: claude -->
  1. Given 验证脚本已创建, When 运行 `npm run validate`, Then 输出当前配置的 Bedrock 模型可用性状态
  2. Given cdk.json 模型预设已定义, When 检查 context, Then 包含至少 3 个常用 Bedrock 模型预设
  3. Given TypeScript 编译通过, When 运行 `npx tsc --noEmit`, Then 0 错误
- **方案**：<!-- source: claude -->
  - 创建 `scripts/validate-bedrock.ts`：使用 AWS SDK v3 调用 ListFoundationModels，验证配置的模型 ID 在目标区域可用
  - 在 `cdk.json` 的 context 中添加模型预设（Claude Sonnet、Claude Haiku 等 Inference Profile 格式）
  - 在 `package.json` 添加 `validate` npm script
- **约束**：
  - 验证脚本需要有效的 AWS 凭证（本地 AWS CLI 配置）
  - 仅验证模型存在性，不测试推理调用

### 执行计划

1. **`scripts/validate-bedrock.ts`**：预部署模型验证脚本
   - 预期：使用 @aws-sdk/client-bedrock 调用 ListFoundationModels，匹配配置的模型 ID，输出验证结果
   - 命令：`npx tsx scripts/validate-bedrock.ts`

2. **`cdk.json` + `package.json`**：模型预设与 NPM 脚本
   - 预期：cdk.json context 含 3+ 模型预设；package.json 含 validate script
   - 命令：`npm run validate`

3. **验证**：端到端验证
   - 预期：tsc 通过 + validate 脚本可执行（无 AWS 凭证时 graceful 降级）
   - 命令：`npx tsc --noEmit && npm run validate`
