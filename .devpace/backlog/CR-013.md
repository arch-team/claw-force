## 关键决策

- **EFS 而非 S3/独立 EBS**：EFS 是最简单的方案 — 天然独立于 EC2 生命周期，NFS mount 透明，不需要应用层改造
- **RETAIN 策略**：EFS 设置 `removalPolicy: RETAIN`，`cdk destroy` 不删除数据
- **手动 ID 回填**：首次部署后用户需将 EFS ID 保存到 `cdk.json`（`efsFileSystemId`），后续部署自动重用
- **公共子网 Mount Target**：默认 VPC 仅有公共子网，EFS mount target 部署到公共子网
