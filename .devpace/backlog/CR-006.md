## 意图

- **用户原话**：我想对 infra 进行优化 <!-- source: user -->
- **范围**：<!-- source: claude, 基于代码审查分析 -->
  - 做什么：
    1. 代码质量：删除死代码（infra-stack.ts）、统一 Props 命名规范、添加 ESLint + Prettier 工具链
    2. 测试补全：为 ALB、WAF、Networking、Compute constructs 添加独立单元测试
    3. 目录结构重构：按 project-structure.md 规范重组（constructs 文件夹化、添加 config/ 环境配置）
    4. CDK Nag 合规：集成 cdk-nag 安全合规检查
  - 不做什么：不改变现有资源逻辑、不修改部署行为、不添加新功能
- **验收条件**：
  1. Given infra-stack.ts 是死代码, When 优化完成, Then 文件已删除且无残留引用
  2. Given 6 个 constructs（networking/iam/compute/alb/waf/monitoring）, When 运行测试, Then 每个 construct 有独立测试文件且全部通过
  3. Given package.json 缺少 lint/format 脚本, When 优化完成, Then ESLint + Prettier 已配置且 `pnpm lint` / `pnpm format:check` 可运行
  4. Given Props 命名不一致（NetworkingProps vs ClawForceXxxProps）, When 优化完成, Then 所有 Props 统一为 ClawForce{Name}Props 格式
  5. Given 无 CDK Nag 集成, When 优化完成, Then cdk-nag 合规检查通过或有合理抑制
  6. Given 所有变更, When 运行 `pnpm test`, Then 全部测试通过
- **方案**：<!-- source: claude, 基于现有代码结构和规范分析 -->
  - 渐进式重构：先清理死代码 → 统一命名 → 添加工具链 → 补全测试 → 目录重构
  - 每步独立可验证，保证中间状态可回退
- **约束**：
  - 不 break 现有 `cdk synth` 和测试
  - 遵循 infra/.claude/rules/ 中的规范
  - 使用 npm（当前项目实际使用 npm，非 pnpm）
