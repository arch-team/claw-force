## 意图

- **用户原话**：在当前已经实现的功能基础上增加一个实际应用场景的验证，比如跟飞书对接 <!-- source: user -->
- **范围**：
  - 做什么：
    1. 在 openclaw.json 动态生成中添加 `channels.feishu` 配置（WebSocket 模式）
    2. 添加 Hooks API 配置（`POST /hooks/agent` 外部集成端点）
    3. 通过 CDK context 传入飞书凭据（`feishuAppId` / `feishuAppSecret`）
    4. 更新 constants、compute construct、stack 和测试
  - 不做什么：不改变 IAM 权限（飞书 WebSocket 出站无需额外权限）、不改变 ALB/WAF/网络配置、不构建独立 Bot 服务
- **验收条件**：
  1. `openclaw.json` 包含 `channels.feishu` 配置（当提供 appId+appSecret 时）
  2. `openclaw.json` 包含 `hooks` API 配置（当飞书启用时自动启用）
  3. 未提供飞书凭据时行为不变（向后兼容）
  4. 108/108 测试通过
  5. lint/format/typecheck/cdk synth 全绿

## 关键决策

- **使用内置飞书插件而非自建**：openclaw-fork 已包含完整的 `extensions/feishu/` 插件（WebSocket/Webhook 双模式、群聊、流式卡片），无需额外开发
- **WebSocket 模式优先**：无需公网回调 URL，适合 EC2 + ALB 部署场景
- **CDK context 传入凭据**：与现有 gateway token 模式一致，后续可升级到 SSM/Secrets Manager
