# OpenClaw Bedrock 模型 ID 缺少 provider 前缀

- **ID**：CR-009
- **类型**：defect
- **严重度**：High
- **产品功能**：Bedrock 模型适配层（PF-002）→ 企业级 AI 员工平台基础设施（BR-001）
- **应用**：infra
- **分支**：fix/openclaw-bedrock-auth
- **状态**：merged
- **复杂度**：S
- **关联反馈**：FB-001

## 缺陷描述

**症状**：用户访问 OpenClaw Gateway 对话界面时，系统返回错误：
```
⚠️ Agent failed before reply: No API key found for provider "anthropic".
Auth store: /home/node/.openclaw/agents/main/agent/auth-profiles.json
(agentDir: /home/node/.openclaw/agents/main/agent).
```

**影响**：AI 对话核心功能完全不可用，平台无法正常服务。

**发现时间**：2026-03-01（REL-002 部署后首次使用）

**环境**：production (i-0bad836a988d3c6fd, us-west-2)

## 根因分析

**根因**：`OPENCLAW_MODEL` 环境变量缺少 `amazon-bedrock/` 前缀。

OpenClaw 通过模型 ID 的前缀判断使用哪个 provider：
- `amazon-bedrock/us.anthropic.claude-...` → 使用 Bedrock provider（IAM 认证，无需 API key）
- `us.anthropic.claude-...`（旧值）→ 默认解析为 Anthropic 直连 → 查找 API key → 报错

**引入点**：CR-002（Bedrock 模型适配层），在 `user-data.ts` 中直接使用了 Bedrock Inference Profile 格式的 model ID 作为 `OPENCLAW_MODEL`，未添加 OpenClaw 要求的 `amazon-bedrock/` 前缀。

**预防措施**：OpenClaw 配置变更应在本地容器环境中端到端验证对话功能，而不仅仅验证部署成功。

## 修复方案

### 实际修复（已完成）

1. **`infra/lib/constructs/user-data.ts`**：`OPENCLAW_MODEL` 从 `${bedrockModelId}` 改为 `amazon-bedrock/${bedrockModelId}`
2. **`infra/assets/openclaw.json`**：添加 `models.providers.amazon-bedrock.region` 配置
3. **`infra/test/user-data.test.ts`**：更新模型 ID 测试 + 新增 Bedrock provider 配置测试

### 部署验证（待用户执行）

需要 `cdk deploy` 重新部署（instance 重建触发新 User Data），或手动 SSH 到现有 instance：
```bash
# 临时修复：修改 .env 中的 OPENCLAW_MODEL
sed -i 's|OPENCLAW_MODEL=us\.|OPENCLAW_MODEL=amazon-bedrock/us.|' /home/ubuntu/openclaw/.env
# 重启容器
cd /home/ubuntu/openclaw && docker compose down && docker compose up -d
```
