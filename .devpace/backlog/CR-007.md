## 意图

- **问题描述**：REL-002 部署验证发现 ALB Target Group 端口与 OpenClaw 实际端口分配不匹配 <!-- source: claude, REL-002 verify -->
- **范围**：
  - 做什么：
    1. 修改 ControlUI TG 目标端口：18790 → 18789（OpenClaw 已将 Control UI 合并到 Gateway 端口）
    2. 评估 Browser TG (18791) 配置：容器内 Browser 服务绑定 127.0.0.1，ALB 无法到达
    3. 更新 docker-compose.yml 端口映射（如需要）
    4. 更新相关测试断言
  - 不做什么：不改变 ALB Listener 规则结构、不修改 WAF 配置、不改变直连模式行为
- **验收条件**：
  1. Given ALB 模式部署, When 访问 ALB DNS, Then Control UI 页面正常返回（非 502）
  2. Given 3 个 Target Groups, When 检查健康状态, Then Gateway 和 ControlUI TG 均为 healthy
  3. Given 端口修改, When 运行 `npm test`, Then 全部测试通过
- **根因分析**：
  - OpenClaw 上游变更：Control UI 从独立端口 (18790) 合并到 Gateway 端口 (18789)
  - Browser 服务绑定 loopback (127.0.0.1) 而非 LAN
  - REL-001 使用直连模式 (enableAlb=false) 部署，未触发此问题
  - REL-002 首次以 ALB 模式在真实环境验证，暴露了端口不匹配
