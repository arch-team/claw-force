## 意图

- **问题描述**：Control UI 聊天页面通过 ALB 访问时出现 "origin not allowed" CORS 错误 <!-- source: user feedback, REL-002 verify -->
- **范围**：
  - 做什么：
    1. 在 `assets/docker-compose.yml` 中配置 `GATEWAY_CONTROL_UI_ALLOWED_ORIGINS` 环境变量
    2. 添加 ALB DNS 地址到允许列表（支持动态获取或配置模板）
    3. 考虑 dev/prod 环境差异（dev 无 ALB，prod 有 ALB）
    4. 更新 user-data.ts 脚本确保配置生效
  - 不做什么：不修改 OpenClaw Gateway 源码、不改变 ALB 配置、不影响直连模式（enableAlb=false）
- **验收条件**：
  1. Given ALB 模式部署, When 访问 http://<alb-dns>/chat?session=main, Then 聊天页面正常工作（无 CORS 错误）
  2. Given 直连模式部署, When 访问 http://<ec2-ip>:18789/chat, Then 聊天页面正常工作
  3. Given 配置变更, When 运行 `npm test`, Then 全部测试通过
- **根因分析**：
  - OpenClaw Gateway 默认 `controlUi.allowedOrigins` 仅允许 gateway host
  - ALB DNS 地址（http://clawforce-alb-1050550383.us-west-2.elb.amazonaws.com）未在允许列表中
  - CORS 预检请求（OPTIONS）被 Gateway 拒绝
  - REL-002 部署成功但 CORS 配置未适配 ALB 场景
